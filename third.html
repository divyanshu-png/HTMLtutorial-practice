Semantic HTML elements provide meaning to the structure of a web page, making it more understandable for both browsers and developers, and improving accessibility and SEO.

1) (header): Represents introductory content, usually containing a group of navigational aids and branding.

Now we will use these layout techniques to enhance our portfolio page.

Under the portfolio page we discuss about three main tags that are used:
1) (header): Contains the main heading of the page, which is "My Portfolio".
2) (main): Contains the main content of the page, including a welcome message, personal introduction, and links to LinkedIn, education profile, and project synopsis. Also includes a section tag providng an overview of what ll I have done as a part of my personal and professional growth!
3) (footer): Contains contact information, including a phone number and email address.

Note: Strong and bold tags i.e b are same and so are emphasis and italics. The curerent text is listed in emphasis tag

Now we will discuss the tags that are included inside the main tag :
1) Section Tag (section): Used to define sections of content, such as "Linkedin", "Education", and "Project, Experience and Certifications". Like for example when we visit the Wikipedia page there are different section to which we can direct, simply by clicking on them.
2) Article Tag (article): Used to define independent, self-contained content, such as a blog post or news article. In this case, it is used in the portfolio page to provide an oveview to the recruiter, and is a common semantic element.
3)Aside tag (aside): Used to display content except the main content i.e like the ads section, call out boxes, author information, realted contents/further readings, bibliographires etc. Basically the content indirectly related to the main content. While the aside tag itself does not apply any special styling, it is typically styled using CSS to visually distinguish it from the main content, such as by placing it in a separate column or adding borders and background colors. 4) Figure and figcaption tags (figure)(figcaption): Used for self-contained content, like images, diagrams, code snippets, with an optional caption.

Common Semantic Tags

This section explores various semantic tags like `header`, `nav`, `main`, `article`, `section`, `aside`, and `footer`.

  • `article`: For self-contained, independent content.
  • `aside`: For tangential content, like sidebars.
  • `footer`: For copyright information, contact details, etc.
